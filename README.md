# Book of Negroes Database Dashboard

A Streamlit web application for searching and exploring records from the Book of Negroes database stored in Azure PostgreSQL.

## Features

- 🔍 **Search Functionality**: Search across all fields or specific columns
- 📊 **Database Statistics**: View table structure and record counts
- � **Demographics Analysis**: Interactive visualizations showing:
  - Race distribution (bar and pie charts)
  - Ethnicity distribution (bar and pie charts)
  - Gender composition (bar and pie charts)
  - Description categories (top 20 descriptions)
- 📥 **Export Results**: Download search results as CSV
- 🎨 **Modern UI**: Clean, responsive interface built with Streamlit and Plotly

## Setup Instructions

### 1. Prerequisites

- Python 3.8 or higher
- Azure PostgreSQL database with Book of Negroes data
- Access to Azure Database for PostgreSQL

### 2. Install Dependencies

```bash
cd database_sofafea_demo
pip install -r requirements.txt
```

### 3. Configure Database Connection

1. Copy the example environment file:
   ```bash
   cp .env.example .env
   ```

2. Configure database connection:
   - Copy `.env.example` to `.env` (it already contains sample values for your Azure PostgreSQL instance)
   - Update the values if they differ from your actual database credentials

### 4. Load Data (Optional)

If you haven't loaded your data yet, use the provided data loading script:

```bash
python load_data.py
```

This script will:
- Read the Excel file generated by your scraper (`../Black_Loyalist_Directory_Consolidated.xlsx`)
- Create the database table if it doesn't exist
- Load all records into PostgreSQL

### 5. Database Table Name

The application uses the table `black_loyalist_directory` which matches your actual database schema. The table contains all the Book of Negroes records with the following column structure:

- ID, Ref_Page, Book, Ship_Name, Commander
- First_Name, Surname, Father_FirstName, Father_Surname, Mother_FirstName, Mother_Surname
- Gender, Age, Race, Ethnicity, Description
- Origination_Port, Origination_State, Departure_Port, Departure_Date, Arrival_Port_City
- Primary_Source_1, Primary_Source_2, Notes

If your table has a different name, update the `table_name` variable in both `app.py` and `load_data.py`.

### 5. Run the Application

```bash
streamlit run app.py
```

The dashboard will be available at `http://localhost:8501`

## Database Schema

The application expects the table `black_loyalist_directory` with the following columns (matching your actual schema):

- **ID** (VARCHAR): Unique identifier for each record
- **Ref_Page** (VARCHAR): Reference page from original source
- **Book** (VARCHAR): Book section (Book One Part One, etc.)
- **Ship_Name** (VARCHAR): Name of the ship
- **Commander** (VARCHAR): Ship commander/master
- **First_Name** (VARCHAR): Person first name
- **Surname** (VARCHAR): Person surname/last name
- **Father_FirstName** (VARCHAR): Father first name
- **Father_Surname** (VARCHAR): Father surname
- **Mother_FirstName** (VARCHAR): Mother first name
- **Mother_Surname** (VARCHAR): Mother surname
- **Gender** (VARCHAR): Gender category (Male, Female, Child Male, Child Female)
- **Age** (VARCHAR): Age at time of record
- **Race** (VARCHAR): Race classification
- **Ethnicity** (VARCHAR): Ethnicity classification
- **Description** (VARCHAR): Detailed description
- **Origination_Port** (VARCHAR): Port of origin
- **Origination_State** (VARCHAR): State/colony of origin
- **Departure_Port** (VARCHAR): Departure port
- **Departure_Date** (VARCHAR): Departure date
- **Arrival_Port_City** (VARCHAR): Arrival port/city
- **Primary_Source_1** (VARCHAR): Primary source reference 1
- **Primary_Source_2** (VARCHAR): Primary source reference 2
- **Notes** (TEXT): Additional notes and raw text

## Troubleshooting

### Connection Issues
- Ensure your Azure PostgreSQL server allows connections from your IP
- Verify firewall settings in Azure Portal
- Check that SSL mode is enabled (required for Azure)

### Search Not Working
- Confirm table name matches `black_loyalist_directory`
- Check column names match the expected schema
- Ensure data types are compatible with ILIKE searches

### Analytics Charts Not Displaying
- Ensure Plotly is installed: `pip install plotly`
- Restart the Streamlit app: `streamlit run app.py`
- Check that the database has data in the race, ethnicity, gender, and description fields

### Performance
- Large result sets may be slow; use the limit option
- Consider adding database indexes on frequently searched columns
- Charts will gradually load as data is fetched from the database

## Security Notes

- Never commit `.env` files with real credentials
- Use Azure AD authentication if possible instead of password
- Consider using Azure Key Vault for secrets in production
